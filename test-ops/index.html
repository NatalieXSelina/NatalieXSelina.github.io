过程中遇到的问题及解决：
jenkins.war启动后，无法访问8080

防火墙问题，解决手段：
sudo systemctl stop firewalld.service #临时关闭防火墙（重启服务器后又会开启的）
systemctl disable firewalld # 永久关闭防火墙（一劳永逸~）

工程中使用到的插件，需要安装一下，非jenkins默认安装的插件

安装 Copy Artifact 插件->fx-code获取前端构建结果
unzip->安装Pipeline Utility Steps插件

前端工程npm install报错

npm install 报错EAI_AGAIN，和dns有关，配置daemon.json解决

{
“dns”: [“10.0.0.2”, “8.8.8.8”]
}

mkdir ./views/pc的时候，第一次需要递归创建。否则会报错

mkdir -p ./views/pc/

后端工程npm install报错（此处感谢老刘，不然我还得排查好久）

是package-lock.json中包的hash不对，重新复制一份即可（但是由于时间间隔久，最新的lock文件已经对不上代码了，重新根据dev创了一份新的）

保存fx-code镜像至服务器的时候，提示磁盘不足，需要进行扩容（早知道一开始就建80G了）

1、在pve上调整磁盘大小

2、扩容sda3：https://www.cnblogs.com/chuanghongmeng/p/16278040.html

3、再扩容centos-root

lvextend -l +100%FREE /dev/mapper/centos-root
xfs_growfs /

根据develop.yml启动后，30001访问（docker run配置的映射端口），报错找不到../fx-code-web/views的资源

看了一下，本地开发的时候，fx-code-pc打包后会生成fx-code-web的静态资源。
然后在fx-code的develop.yml（本地开发配置）中定义了路径

static:
public: ../fx-code-web/dist
views: ../fx-code-web/views
这边浪费了蛮多时间，一开始以为需要再针对fx-code-web工程处理一下。一个上午都在看怎么在不构建web工程的情况下，把静态资源放进去。

后来在刘大哥的提醒下，仔细看了一下针对2个环境，前端webpack打包的区别，发现本地开发的时候，是专门复制了一份web资源出去，所以才在本地看到了fx-code-web的目录，但是dev环境的是全部打到web.zip下，只是本地开发的那份yml的static设置是基于前者的，将其调整一下即可

static:
public: ./public
views: ./views
