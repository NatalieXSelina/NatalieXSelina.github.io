chai-regular expression

Author: NatalieXSelina Date: April 20, 2023  14:52:02

背景：

现象：导入重名校验自动化用例失败

调查原因：后端返回顺序变动，可能是数据库索引变更，但是排序是由前台完成的，因此顺序变动可能是常态

fix：避免后端返回顺序变动导致失败，使用正则匹配

一些前置知识：js正则匹配  |  chai断言库(主要讲的是各种相等、包含)  |  chai中match断言的使用 

由前置知识可得：

正则匹配需要使用(x|y)
相等、包含等断言方法在本次场景中无法完成正则匹配（需要字符串）
代码中需要改用match断言
=>具体代码中：

1、先使用toString方法将数组转为字符串

2、进行正则匹配

新代码

        cy.task('readExcelFile', dateString).then((list) => { // 下载的重名情况Excel文件名称包含当前年月日
            expect(list[0]).to.deep.equal(['重名部门/成员', '所处部门', '部门/成员id', '建议导入样式'])
            // 避免后端返回顺序变动导致失败，使用正则匹配
            expect(list[1].toString()).to.match(/重名部门,内部组织\/ExcelTest,(3|4),重名部门#(3|4)#/)
            expect(list[2].toString()).to.match(/重名部门,内部组织\/ExcelTest,(3|4),重名部门#(3|4)#/)
            expect(list[3].toString()).to.match(/重名成员,内部组织\/ExcelTest,(0001|0002),重名成员#(0001|0002)#/)
            expect(list[4].toString()).to.match(/重名成员,内部组织\/ExcelTest,(0001|0002),重名成员#(0001|0002)#/)
            expect(list[5].toString()).to.match(/exceltest,内部组织\/ExcelTest,(999|jdy-azceqq08krlv),exceltest#(999|jdy-azceqq08krlv)#/)
            expect(list[6].toString()).to.match(/exceltest,内部组织\/ExcelTest,(999|jdy-azceqq08krlv),exceltest#(999|jdy-azceqq08krlv)#/)
        })

旧代码

        cy.task('readExcelFile', dateString).then((list) => { // 下载的重名情况Excel文件名称包含当前年月日
            expect(list[0], 'header').to.deep.equal(['重名部门/成员', '所处部门', '部门/成员id', '建议导入样式'])
            expect(list[1], 'content').to.deep.equal(['重名部门', '内部组织/ExcelTest', '3', '重名部门#3#'])
            expect(list[2], 'content').to.deep.equal(['重名部门', '内部组织/ExcelTest', '4', '重名部门#4#'])
            expect(list[3], 'content').to.deep.equal(['重名成员', '内部组织/ExcelTest', '0001', '重名成员#0001#'])
            expect(list[4], 'content').to.deep.equal(['重名成员', '内部组织/ExcelTest', '0002', '重名成员#0002#'])
            expect(list[5], 'content').to.deep.equal(['exceltest', '内部组织/ExcelTest', '999', 'exceltest#999#'])
            expect(list[6], 'content').to.deep.equal(['exceltest', '内部组织/ExcelTest', 'jdy-azceqq08krlv', 'exceltest#jdy-azceqq08krlv#'])
        })
